# 试过十几个翻译插件后，我为什么只留下这个「只做中英」的？

<p align="center">
  <img src="../../public/icon-128.png" width="96" height="96" alt="Lingua Bridge"/>
</p>

每天读英文文档、刷技术博客、看 YouTube，最烦的不是「看不懂」，而是**流程太碎**：

选中 → 复制 → 打开百度/谷歌翻译 → 粘贴 → 再切回来继续看。  
视频更糟：开着自动字幕，还是要盯着英文，或再开一个同传工具。

**Lingua Bridge** 解决的就是这件事：把「中英互译」嵌进浏览本身——划词、整页、视频同传，一个扩展搞定，不用来回拷贝。

---

## 它是什么

Lingua Bridge 是 Chrome / Edge / Firefox 浏览器扩展，**刻意只做中英互译（Chinese ↔ English）**。

核心能力三块：

1. **划词气泡**：选中即译，原文/译文对照，还能看词条讲解（音标、释义）
2. **可视区整页翻译**：当前屏幕里的文字一键译完，可随时恢复原文
3. **视频同声传译**：可拖动的双语字幕；需要时还能语音朗读译文

---

## 一图看懂：划词就懂，不用复制粘贴

打开主开关后，在网页选中中文或英文，旁侧出现气泡：点 **Translate** 立刻出译文，还可一键 **Full page**。

![划词气泡：英文译为中文，附音标与释义](../images/selection-bubble.png)

上图里，选中 *Simultaneous interpretation*，立刻得到「同声传译」，并附带音标与简要释义——这就是「页面内完成」的可用性：上下文不丢、窗口不用切。

---

## 控制台一眼看清状态

工具栏弹窗是控制中心：主开关、实时状态（供应商、同传样式、本页是否同传、是否检测到视频），以及同传输出模式。

![Lingua Bridge 弹窗：实时状态与同传控制](../images/popup.png)

不用猜「现在到底开没开」：状态区把方向、供应商、视频检测、同传是否运行写清楚；底部还有一行摘要。这是很多「功能很多、状态很糊」的插件做不到的。

---

## 视频同传：静默字幕 or 语音传译

在有视频的页面，选好样式后开启同传：

- **静默字幕** — 双语浮层（可拖动、历史、折叠/关闭）
- **语音传译** — 朗读译文，同时仍显示字幕

![视频同传双语字幕浮窗](../images/si-caption.png)

关字幕浮窗会同步关闭同传，并更新弹窗按钮与实时状态——不会留下「幽灵进程」。对天天看英文演讲、技术分享的人，这一步往往比「再装一个同传客户端」省太多。

---

## Key 可选 + 本机安全

没有 API Key 也能先用免费路径；配置 Key 后 AI 讲解与视频音轨同传更强。供应商可选 OpenAI、DeepSeek、讯飞、OpenRouter、Anthropic、自定义 OpenAI 兼容网关。

![设置页：多供应商、凭证本机存储、可选加密保险库](../images/options.png)

关键点：**无自建后端**。Key 存在你本机浏览器扩展存储；还可开启 PBKDF2 + AES-GCM 会话保险库。对在意隐私、不愿把密钥交给第三方翻译 SaaS 的人，这是硬优势。

## 可用性：为什么「真的省事」

| 痛点 | 常见做法 | Lingua Bridge |
|------|----------|---------------|
| 读网页 | 复制到外部翻译站 | 页面内划词 / 整页，零粘贴 |
| 查词学习 | 再开词典站 | 气泡里直接看讲解 |
| 看视频 | 英文字幕硬啃 / 另装同传 | 同扩展开双语字幕或语音传译 |
| 方向选择 | 手动选「英→中」 | 自动识别中英，互译到另一侧 |
| 入门门槛 | 必须先配 Key | **无 Key 也能用**；有 Key 再增强 |

典型路径：

1. 打开主开关  
2. 划词点翻译，或点整页  
3. 视频页选「静默字幕」或「语音传译」→ 开启同传  
4.（可选）设置里填自己的 API Key  

**默认不是整页狂刷翻译**，而是你需要时再译——干扰少，更符合「边读边用」。

## 产品优势（相对「能翻译」这件事本身）

**1. 只做中英，反而更稳**  
多语种插件什么都想覆盖，识别、术语、TTS 往往「样样通、样样平」。Lingua Bridge 把能力压在中英一对上，日常更省心。

**2. 浏览 + 视频同传一体**  
划词讲解、整页翻译、视频字幕/语音传传，同一扩展；状态跟弹窗联动。

**3. Key 可选，不是「有 Key 才能活」**  
无 Key 可起步；有 Key 解锁更强 AI 与视频音轨同传。能力分层清晰。

**4. 无后端、密钥本机**  
密钥不经本项目服务器；可选本机加密加固。

## 和其他方案比，差在哪

| 类型 | 你付出的成本 | Lingua Bridge 的相对优势 |
|------|--------------|--------------------------|
| 百度/谷歌翻译网页版 | 复制粘贴、切标签 | **告别拷贝**，上下文不丢 |
| 浏览器内置整页翻译 | 快，但缺划词讲解、视频同传 | 学习向讲解 + 视频 SI + 多供应商 |
| 多语种「沉浸式」插件 | 功能多、设置复杂 | **中英专用**，路径更短 |
| 纯划词词典 | 查词强，整页/视频弱 | 查词 + 整页 + **视频同传** |
| 独立同传软件 | 再装一个、再配一套 | 浏览器里直接开，和划词同习惯 |

一句话：  
你要的是「把英文网页/视频变成可扫读的中英体验」，不是「全球 100 种语言都能译一点」——那它就比「万能翻译插件」更对口。

---

## 适合谁 / 不适合谁

**适合：** 中英文档与新闻读者、跨境内容消费者、边看边学英语的人、已有 OpenAI/DeepSeek/讯飞等 Key 的进阶用户。

**不适合：** 需要日语、韩语、小语种互译的人——产品刻意不做，这是定位，不是漏功能。

---

## 结语

真正好用的翻译工具，不是菜单最多的那个，而是**让你少做一步，还把上下文留在页面上**。

Lingua Bridge 的取舍很清楚：只做中英、页面内完成、视频也能同传、无 Key 可起步、有 Key 可增强、密钥不经项目后端。

如果你也受够了「复制 → 粘贴 → 再切回来」，值得装上试一天——看会不会再也回不去旧流程。
